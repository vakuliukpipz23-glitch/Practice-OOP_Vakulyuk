# Practice-OOP_RPG

Компактний консольний RPG-симулятор для навчальної практики з об’єктно-орієнтованого програмування.

## Структура проєкту

- `Domain/PracticeOOP_RPG.Domain` — доменні моделі, ієрархія персонажів, зброя та бойова логіка.
- `App/PracticeOOP_RPG.App` — консольна програма для демонстрації бою.
- `Tests/PracticeOOP_RPG.Tests` — юніт-тести для перевірки логіки бою та поведінки патернів.

## Запуск

1. Відкрити кореневу папку `Practice-OOP_Vakulyuk`.
2. Виконати `dotnet run --project App/PracticeOOP_RPG.App/PracticeOOP_RPG.App.csproj`.
3. Виконати тести: `dotnet test`.

## Особливості

- Впроваджено `Strategy` для різних типів атак (melee та spell).
- Впроваджено `Factory` для створення типових ворогів та зброї.
- Додано систему рівнів та досвіду для гравця з автоматичним підвищенням характеристик.
- Реалізовано управління ресурсами: мана та еліксири для відновлення HP/MP.
- Використано події для логування бою (`Observer`-подібна модель).
- Додано `Repository<T>` і `Cache<TKey, TValue>` для типобезпечних колекцій.
- Реалізовано збереження і завантаження стану гри у форматах JSON та XML.
- Реалізовано `extension methods` для `ForEach`, `Map`, `Reduce`.
- Додано `RetryPolicy` та обробку винятків з українськими повідомленнями.
- Покрито юніт-тестами, зокрема з використанням `Moq`.
- Додано UML-діаграму: `UML_ClassDiagram.puml`.

## Стан проєкту

Виконано всі основні етапи навчальної практики: аналіз предметної області, UML, інкапсуляція, наслідування, інтерфейси, узагальнені типи, делегати, LINQ, патерни, серіалізація, тести та документація.
